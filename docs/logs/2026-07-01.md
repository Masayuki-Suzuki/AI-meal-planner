## 2026-07-01 — Day 1: 技術スタック選定

- TypeScript: 現在のフロントエンド開発では実質必要不可欠。静的型付けにより
  バグを開発段階で早期に検出でき強力。最新ES機能や将来追加予定の機能を先取りして
  活用でき、DX向上も狙える。

- Next.js: 主流になりつつある React Server Components をフルに活用するための
  ほぼデファクトのフレームワーク。RSCの実務知見を積む目的で採用。React単体からの
  上積みとしてスキルセット上も強力。

- App Router: Page Routerに比べ歴史は浅いが、新規プロジェクトの推奨として定着。
  RSCを前提に設計されており、ルーティング段階で loading state や fetch を
  組み込めるのが強力。RSCをフルに使うには App Router が必要なため採用。

- TailwindCSS: スタイリングの労力を極限まで減らせ、汎用性・自在性が高く、
  個人的に使い慣れているため採用。クラス名が長くなりがちな点が欠点だが、
  近年の開発では重要視されない傾向のため許容する。

- 後で疑う点: App Router の安定性・推奨状況は公式ドキュメントで最新を確認する
  （このログ執筆時点の認識が古い可能性大）
